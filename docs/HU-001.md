# HU-001: Inicio de sesión con correo y contraseña

## Capa 1 — Intención (el "qué" y el "para qué")
Como usuario registrado quiero iniciar sesión con mi correo y contraseña
para acceder a mi panel de forma segura.

## Capa 2 — Criterios de aceptación (lo verificable)
- **CA1 — Éxito:** credenciales válidas → `{ ok: true, panel: ... }` y se reinician los intentos fallidos.
- **CA2 — Fallo:** credenciales inválidas → `{ ok: false }` con mensaje genérico que NO revela si falló el correo o la contraseña.
- **CA3 — Bloqueo:** tras 5 intentos fallidos la cuenta se bloquea 15 minutos; durante el bloqueo siempre devuelve el error genérico.
- **CA4 — Auditoría:** cada intento (éxito o fallo) se registra SIN la contraseña.

## Capa 3 — No funcionales / restricciones
- Contraseñas cifradas con **bcrypt**; nunca comparadas ni guardadas en texto plano.
- Ningún dato sensible (contraseña, token, datos personales) en logs ni en el repo.
- Python 3.11 + pytest. Toda función pública lleva su test.
